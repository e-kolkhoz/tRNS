# Тест: гипотеза "DAC не нужен — синус через I2S PDM TX"

## Цель
Проверить, что встроенный I2S PDM TX модулятор ESP32-S3 + простой ФНЧ
способен дать чистый сигнал амплитудой 1–1.5 В без внешнего DAC,
И что архитектура "loop buffer на канал" покрывает все режимы стимуляции.

## Каналы (для текущего теста)
* ch1 = GPIO1 — синус 640 Гц,    amplitude 1.0 В
* ch2 = GPIO2 — синус 64 000 Гц, amplitude 1.5 В
* offset на обоих 1.65 В (середина PDM, фиксированно — это не настройка)

## Sample rate
Одна общая (I2S PDM stereo). Стартуем с **384 kHz** (≥ 6× для 64 kHz).
Если драйвер не подымет — спускаемся: 256k → 192k. Фиксируем потолок.

PDM clock внутри будет десятки MHz → шум модулятора уносится далеко
вверх, пассивный RC ФНЧ ~150 kHz фильтрует с большим запасом.

## Архитектура: loop buffer на канал

Каждый канал имеет независимый предрасчитанный кольцевой буфер ровно
в один период сигнала. Длина = `sample_rate / signal_freq`. Натурально
покрывает все режимы:

| Режим      | Что лежит в буфере                          | Длина                        |
|------------|---------------------------------------------|------------------------------|
| tACS       | один период синуса                          | sample_rate / freq           |
| tRNS       | один период полосового шума                 | sample_rate / lower_freq     |
| tDCS       | константа                                   | 1 семпл                      |
| Custom WAV | загруженный пресет                          | длина WAV в семплах          |

Структура канала:

```c
struct Channel {
    const int16_t* loop;       // нормализованный шаблон, ±INT16_MAX = ±1.0
    uint32_t       loop_size;  // в семплах
    uint32_t       pos;        // текущая позиция, обнуляется по wrap

    int32_t        gain_q15;        // текущее усиление, 0..32768 (=1.0)
    int32_t        target_gain_q15; // куда едем (для fade)
    int32_t        gain_step_q15;   // приращение/семпл; 0 = без ramp
};
```

* Буфер всегда **нормализованный** (±INT16_MAX). Амплитуда —
  отдельная переменная `gain_q15`, применяется при копировании в DMA:
  `dma_sample = (loop[pos] * gain_q15) >> 15`.
  Это даёт смену амплитуды без пересчёта буфера и встроенный hook
  для плавного fade в будущем.
* **Смена частоты** — пересчёт нового буфера в фоне + атомарная замена
  указателя `loop` (double-buffer). Никаких разрывов фазы.
* **Смена амплитуды** — одна запись в `target_gain_q15`. Если включён
  ramp (`gain_step_q15 != 0`) — поедет плавно, иначе моментально.
* **Смена режима** (синус → шум → WAV) — то же самое: пересчитали
  буфер, поменяли указатель.

## Writer task (core 1)

```
loop:
  заполнить N стерео-семплов в локальный chunk:
    for each sample:
      s1 = (ch1.loop[ch1.pos++ % ch1.loop_size] * ch1.gain_q15) >> 15
      s2 = (ch2.loop[ch2.pos++ % ch2.loop_size] * ch2.gain_q15) >> 15
      // (если когда-нибудь включим ramp: пододвинуть gain_q15 → target)
  i2s_channel_write(handle, chunk, ..., portMAX_DELAY)
```

`i2s_channel_write` — блокирующий, IDF сам разруливает back-pressure
по DMA-кольцу. Никаких ISR руками ловить не надо.

## Почему DMA НЕ натягиваем на loop напрямую

Хотелось бы — нет копирования. Но:
1. Два канала с разной длиной лупа → одно DMA-кольцо физически не
   совпадает с обоими лупами.
2. Применение gain (амплитуда + ramp) требует промежуточной обработки.

Копирование в DMA — это просто индексация и shift, дешевле некуда.
Никаких mul/div/sin() в горячем пути.

## Амплитуда / offset (электрика)

* PDM out 0…3.3 В, среднее (после RC) при `int16_t = 0` = 1.65 В.
* Полный размах int16 → ±1.65 В.
* gain_q15 = 32768 → пик ±1.65 В.
* 1.0 В amplitude → gain_q15 ≈ 19880.
* 1.5 В amplitude → gain_q15 ≈ 29818.
* (Финальное значение калибруется по осциллографу, в железе ещё ОУ
  и делитель — это вне прошивки.)

## Внешняя обвязка (на железе)
* RC ФНЧ ~150 kHz cutoff на каждый канал (R=1k, C=1nF — ориентир).
* Опционально буферный rail-to-rail ОУ.

## План текущего шага
1. Модуль `tone_gen` (ToneGen):
   * `begin(sample_rate)` — инициализация I2S PDM TX, запуск writer task.
   * `setSineLoop(ch, freq_hz, gain_q15)` — пересчёт sine-лупа,
     атомарная замена.
   * `setGain(ch, gain_q15)` — мгновенно.
   * (запас под будущее: `setLoop(ch, int16_t* data, size_t n, gain_q15)`,
     `rampTo(ch, gain_q15, ms)`.)
2. В `setup()` после USBFlash::mount(): `ToneGen::begin(384000)`,
   две `setSineLoop()`.
3. Прошить, померить осциллографом GPIO1/GPIO2 через ФНЧ.
4. Если 384k не стартует — пробуем 256/192k, фиксируем потолок.

## Что в архитектуру заложено заранее (не реализуем сейчас, но не сломается)
* Произвольный источник лупа (sine, шум, WAV) — все идут в один и тот
  же `setLoop(...)`.
* Плавный fade in/out — через `gain_step_q15` (поле уже в Channel).
* Смена частоты на лету — через double-buffer + atomic pointer swap.
